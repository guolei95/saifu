# 赛赋（SaiFu）—— 项目方案框架

> 搭建时间：2026-05-06
> 竞赛类型：互联网+/大创方向
> 用途：拿这个框架和导师/队友对齐，10分钟拉通理解

---

## 项目标语

> **赛者非异也，善假于赋也。**
>
> 你不是不够优秀，只是缺少个性化的赛事信息、精准的备赛资源、得力的AI工具。而赛赋，就是你"善假于物"的核心载体。

---

## 一句话逻辑线

> 双非学生竞赛参与率低，不是不想参加，而是「不知道有什么」和「不知道怎么准备」→ 现有竞赛平台只做信息聚合，不做AI全流程辅助 → 赛赋做覆盖选赛→备赛→方案→路演的AI全流程平台 → 让每个学生都有底气参赛。

### 第一章：项目概述（建议占比 8%）

**核心问题**：这是什么项目？一句话能说清楚吗？

1.1 项目名称：赛赋（SaiFu）—— AI驱动的竞赛全流程辅助平台
1.2 一句话定位：帮大学生从「不知道能参加什么比赛」到「拿得出手的参赛方案」的AI助手
1.3 核心价值：信息差填平 × 备赛流程AI化 × 方案输出专业化
1.4 愿景：让每个想参赛的学生，都有AI做队友

### 第二章：需求分析（建议占比 15%）

**核心问题**：我们要解决的问题真的存在吗？有多严重？

2.1 目标用户画像：双非大一到大三学生，有意愿参赛但缺乏信息和方法
2.2 四大核心痛点（调研已证实）：

- 信息差：班级群只有志愿者通知，竞赛靠人脉口口相传
- 分不清：不知道什么比赛对保研/考研有帮助
- 不匹配：专业课没学完，怕乱报浪费时间
- 组队难：想参加但找不到志同道合的队友
2.3 现有方案缺陷：赛氪10年只做信息聚合，无AI功能；竞品功能初级
2.4 需求验证路径：100份问卷 + 10人深度访谈（📊 待执行）
2.5 需求优先级：选赛推荐 > 方案写作辅助 > 备赛规划 > 组队匹配

篇幅：约1500字 | 建议负责人：商科/社科队友 ⚠️ 待招募

↓ 逻辑承接：本章证明「问题真实 + 没人解决好」→ 第三章回答「我们怎么解决」

### 第三章：产品设计（建议占比 20%）

**核心问题**：赛赋具体做什么？用户怎么用？

3.1 整体架构：三方向 + 通用层

- 🏗️ 大创方向：选赛 → 调研 → 方案设计 → PPT路演 → 赛后沉淀
- 💼 商赛方向：同上流程，调研/方案侧重商业分析
- 📊 数模方向：备赛规划 → 学习刷题 → 模拟实战 →（建模路线+论文）
3.2 入口双路径：
- 路径A：「我知道要参加什么」→ 直接进入对应方向流程
- 路径B：「我不知道能参加什么」→ 多维打分推荐 → 用户选择 → 进入流程
3.3 核心功能（每项一句话）：
- 选赛推荐：专业+兴趣+年级+可操作性，四维打分匹配
- 调研辅助：往届作品拆解 + 市场行业分析 → 自动生成调研报告
- 方案生成：框架搭建 → 分章撰写 → AI评审 → 审改迭代
- PPT路演：结构优化 + 模拟答辩 + 演讲稿打磨
- 赛后沉淀：复盘 → 经验提取 → 简历素材化
3.4 产品形态：Web应用 + 微信小程序（MVP阶段先做网页版）

篇幅：约2000字 | 建议负责人：小雷（产品设计）

↓ 逻辑承接：说清「产品长什么样」→ 第四章回答「凭什么做得出来」

### 第四章：技术方案（建议占比 12%）

**核心问题**：技术怎么做？壁垒在哪？

4.1 技术架构：前端（Web/小程序）→ 后端API → AI引擎层 → 知识库/提示词库
4.2 核心AI能力：大语言模型 + RAG检索增强（竞赛知识库）+ 结构化提示词体系
4.3 知识库构建：赛事库（300+赛事信息）+ 方案模板库 + 往届获奖案例库
4.4 技术壁垒：

- 提示词体系：16个竞赛场景垂直优化提示词（非通用AI能替代）
- 竞赛知识库：结构化赛事数据+方案模板，持续积累形成数据飞轮
- 用户数据飞轮：用户越多 → 方案库越丰富 → 推荐和辅助越精准

### 第五章：市场分析（建议占比 10%）

**核心问题**：市场有多大？对手在做什么？

5.1 市场规模：全国3000+高校，双非院校2000+所，在校生超3000万（⚠️ 待精确核实）
5.2 唯一对标：赛氪——820+高校覆盖，但无AI功能、体验差
5.3 竞品空白：「AI驱动的竞赛全流程辅助」目前未查到省级以上获奖案例
5.4 差异化：

- 赛氪做信息聚合 → 赛赋做AI全流程
- 不做通用AI工具 → 竞赛场景垂直优化
- 从双非视角出发 → 做"从不敢想到敢于开始"的第一公里
5.5 黄石本地政策红利：国家数字化转型试点城市 + AI城市建设行动计划 + 创业补贴5000元

### 第六章：商业模式（建议占比 10%）

**核心问题**：谁付钱？怎么赚钱？

6.1 用户分层：

- C端免费：选赛推荐 + 方案框架 + 通用备赛指导
- C端付费：深度方案撰写 + AI评审迭代 + 模拟答辩
- B端合作：高校双创学院/就业指导中心采购
6.2 收入模型：会员订阅 + 单次深度服务 + 高校B端合作
6.3 获客路径：校内社群 → 双非高校联盟 → 竞赛社区KOL合作
6.4 成本结构：大模型API调用（主成本）+ 服务器 + 知识库维护
6.5 为什么付钱：对标ChatGPT Plus模式——免费能用，深度付费

篇幅：约1000字 | 建议负责人：商科队友 ⚠️ 待招募

### 第七章：实施计划（建议占比 10%）

**核心问题**：怎么做？什么时候做完？

7.1 MVP阶段（1-2个月）：

- 调研验证（问卷+访谈，数据写入需求分析章）
- 网页MVP搭建（静态HTML展示核心流程，Mock数据）
- 软著申请启动（受理通知书1-2个月到手）
7.2 迭代阶段（3-4个月）：
- 方案设计模块完整版（框架+分章撰写+AI评审）
- 竞赛知识库初版（100+赛事信息）
- 用户内测 → 反馈迭代
7.3 完善阶段（5-6个月）：
- 选赛推荐 + 商赛/数模方向扩展
- PPT路演模块 + 模拟答辩上线

### 第十章：风险与应对（建议占比 5%）

**核心问题**：最可能失败在哪？怎么防？

10.1 技术风险：大模型API调用成本高 → 初期免费模型 + 提示词优化降低token消耗
10.2 竞争风险：赛氪跟进AI功能 → 靠垂直场景提示词壁垒 + 用户数据飞轮
10.3 执行风险：团队不全 → 先MVP验证，同步招募队友
10.4 合规风险：竞赛AI使用规范 → 主动提醒用户标注AI使用，不做代写
10.5 商业化风险：学生付费意愿待验证 → MVP阶段免费策略验证留存
