## 一、项目背景（给 AI 读的上下文）

### 1.1 我们在做什么

把现有的 Python CLI 竞赛匹配脚本（`0503-match_competitions.py`）改造成一个**在线网页平台**，让零基础用户打开浏览器就能用。

### 1.2 核心原则

- **用户零门槛**：打开网址 → 填画像 → 点按钮 → 看结果。不需要安装、不需要 API Key、不需要配置环境
- **后端持有 API Key**：DeepSeek Key 只在服务器环境变量里，前端代码里不能出现
- **架构可扩展**：现在只做匹配，但代码结构要能以后加竞赛浏览、调研库、方案助手
- **面向中国用户**：前端放 Vercel（香港节点），后端放 Railway（美国），延迟可接受

### 1.3 项目文件路径

```
工作区根目录：D:\我的竞赛项目-AI赋能竞赛系统\
现有核心文件：
  - D:\我的竞赛项目-AI赋能竞赛系统\02-方案\0503-match_competitions.py  ← 现有匹配逻辑（约2000行）
  - D:\我的竞赛项目-AI赋能竞赛系统\03-素材\用户画像表单.html              ← 现有前端表单
  - D:\我的竞赛项目-AI赋能竞赛系统\项目信息.md                            ← 项目信息
  - D:\我的竞赛项目-AI赋能竞赛系统\进度追踪.md                            ← 进度
  - D:\我的竞赛项目-AI赋能竞赛系统\项目日志.md                            ← 日志
```

### 1.4 现有代码的核心功能（AI 要先读懂）

`0503-match_competitions.py` 做的事：
1. 接收用户画像（学校、专业、年级、兴趣、技能等）
2. 根据画像用 DeepSeek 生成搜索查询词
3. 用 DuckDuckGo 搜索竞赛信息
4. 用 DeepSeek 匹配分析（打分、匹配原因、避坑提醒）
5. 用内置常识库交叉验证（防止 AI 幻觉）
6. DeepSeek 自审查（合理性二次检查）
7. 输出格式化结果

## 二、技术选型（已确定，不要改）

| 层 | 技术 | 理由 |
|----|------|------|
| 后端框架 | **FastAPI**（Python） | 轻量、能把现有代码几乎原封不动变 API |
| 前端 | **纯 HTML + CSS + JavaScript** | 你现有表单已经是 HTML，直接扩展，不推翻 |
| 数据库 | **SQLite**（可选，第一步可不用） | 后续缓存竞赛数据用，第一步可能不需要 |
| 后端部署 | **Railway**（railway.com）免费档 | 支持 Python，部署简单 |
| 前端部署 | **Vercel**（vercel.com）免费档 | 有香港节点，国内访问快 |
| AI 服务 | DeepSeek API（你现在的 Key） | 不变 |

### 任务 2：创建后端核心模块 — config.py

**目标**：统一管理配置，从环境变量读 API Key。

**产出文件**：`backend/config.py`

**要求**：
- 从 `os.environ` 读取 `DEEPSEEK_API_KEY`
- 设置默认值（模型名称、搜索数量等）
- 如果 Key 缺失，启动时报错提示

**代码骨架**：
```python
import os

DEEPSEEK_API_KEY = os.environ.get("DEEPSEEK_API_KEY")
if not DEEPSEEK_API_KEY:
    raise RuntimeError("请在环境变量中设置 DEEPSEEK_API_KEY")

DEEPSEEK_BASE_URL = "https://api.deepseek.com"
DEEPSEEK_MODEL = "deepseek-chat"
MAX_SEARCH_RESULTS = 10
```

---

### 任务 3：创建后端核心模块 — ai_client.py

**目标**：封装 DeepSeek API 调用，让其他地方直接调函数。

**产出文件**：`backend/services/ai_client.py`

**要求**：
- 一个通用函数 `call_deepseek(messages, temperature=0.7, max_tokens=4096) -> str`
- 一个封装函数 `call_deepseek_json(messages, ...) -> dict`（自动解析 JSON 返回）
- 使用 OpenAI 兼容 SDK（`from openai import OpenAI`，因为 DeepSeek 兼容 OpenAI 接口）
- 从 `config.py` 读 Key 和 base_url

**目标**：封装 DuckDuckGo 搜索。

**产出文件**：`backend/services/search.py`

### 任务 5：创建后端核心模块 — knowledge_base.py

**目标**：把现有代码里的常识库（硬编码的竞赛数据）提取出来，集中管理。

**产出文件**：`backend/services/knowledge_base.py`

**要求**：
- 从现有代码中提取所有硬编码的竞赛信息（竞赛名称、适合专业、时间规律、级别等）

- 提供 `validate_competition(name: str) -> dict | None` 函数（查常识库确认竞赛是否存在）
- 标注数据来源（如 `⚠️ 基于公开信息整理，可能有延迟`）

---

### 任务 6：创建后端核心模块 — engine.py

**目标**：把现有代码的主流程逻辑整理成可调用的函数。

**产出文件**：`backend/match/engine.py`

**要求**：
- 主函数：`async def match_competitions(profile: dict) -> dict`
- 输入 profile 格式：
  ```python
  {
      "school": "北京大学",
      "major": "计算机科学",
      "grade": "大二",
      "interests": ["AI", "创业"],
      "skills": ["Python", "数据分析"],
      "prefer_type": "",  # 个人/团队/不限
      "prefer_level": "",  # 校级/省级/国家级/不限
      "available_time": ""  # 课余/周末/假期
  }
  ```
- 处理流程（从现有代码提取）：
  1. 接收用户画像
  2. 用 DeepSeek 生成搜索查询词
  3. 并行搜索（多个查询词同时搜）
  4. 搜索结果汇总去重
  5. 用 DeepSeek 匹配分析（为每个竞赛打分、写理由）
  6. 常识库交叉验证（过滤不存在的竞赛、纠正错误信息）
  7. DeepSeek 自审查
  8. 格式化返回结果
- 返回格式：
  ```python
  {
      "success": True,
      "competitions": [
          {
              "name": "竞赛名称",
              "score": 85,  # 匹配分数 0-100
              "match_reasons": ["原因1", "原因2"],
              "level": "国家级",
              "type": "学科竞赛",
              "register_time": "2026年5月",
              "fee": "免费",
              "format": "团队(3-5人)",
              "tips": "备赛建议/避坑提醒",
              "related_cases": "相关案例参考"
          }
      ],
      "total_count": 5,
      "search_queries_used": ["搜索词1", ...],
      "ai_warnings": ["AI 分析提示..."]
  }
  ```

### 任务 7：创建 FastAPI 入口 — main.py

**目标**：把 engine.py 变成一个 Web API。

**产出文件**：`backend/main.py`

**要求**：
- FastAPI 应用
- CORS 中间件（允许前端跨域请求）
- 端点：`POST /api/match`
  - 接收 JSON：用户画像
  - 调用 `match_competitions(profile)`
  - 返回匹配结果
- 端点：`GET /api/health`
  - 返回 `{"status": "ok"}`
- 启动命令：`uvicorn main:app --host 0.0.0.0 --port $PORT`
