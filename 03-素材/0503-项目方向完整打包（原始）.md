## 一、项目核心定位

### 要解决的问题

**双非高校学生面临的信息不对称困境：**
- 班级群只发水活动（观众、志愿者），不发高含金量竞赛
- 学生想参加竞赛但不知道有什么、什么时候报名、适不适合自己
- 学校竞赛信息散落在N个官网/公众号/PDF里
- 黄石本地企业有项目需求但学生不知道
- 政府有创业补贴政策但没人告诉你

**这不是一个人的问题——是所有双非学校学生的系统性困境。**

### 核心洞察

把「官方语言」翻译成「学生能听懂的话」：

| 网上已有的信息 | 学生需要的信息 |
|-------------|-------------|
| 「第十八届挑战杯全国大学生课外学术科技作品竞赛预通知」 | 「我大一，网络工程，这东西我能参加吗？什么时候报名？要准备什么？」 |
| 「PCB产业供应链协同数字化改造」 | 「我一个写代码的学生，能帮企业做什么？」 |
| 「黄石市中小企业数字化转型试点城市实施方案」 | 「这里有什么钱我可以申请？什么条件？」 |

AI可以做这个翻译，但没人做。

## 二、对标参考：「面立方」——双非高校全国特等奖

学校：武昌理工学院（双非）
奖项：第十九届"挑战杯"全国大学生课外学术科技作品竞赛"揭榜挂帅"擂台赛 **人工智能领域全国特等奖**（2025年）
项目内容：深度融合多模态技术，构建智能评测体系，实现企业高效选才与求职者精准成长的双向赋能

### 「面立方」不是提示词，是一个有7层的系统产品

| 层次 | 普通AI工作流 | 「面立方」产品化 |
|------|------------|---------------|
| 入口 | 打开AI对话框 | 独立网站/小程序，任何人都能访问 |
| 交互 | 打字输入 | 支持语音、视频录制、简历上传 |
| 评测 | AI给文字反馈 | 多模态分析——语音语调、微表情、回答逻辑、语速，生成评分雷达图 |
| 数据 | 聊完就没了 | 保存每次面试记录，追踪进步曲线 |
| 题库 | 临时想问题问AI | 按行业/岗位/难度分类的结构化题库 |
| 闭环 | 凭感觉判断 | 量化评分 + 具体改进建议 + 薄弱点标注 |
| 壁垒 | 零 | 1项专利 + 1项软著 + 4篇论文 + 企业合作意向书 |

### 双非高校拿奖的核心规律

三个双非学校的成功案例：

| 获奖项目 | 学校 | 奖项 | 共性 |
|---------|------|------|------|
| 面立方 | 武昌理工学院 | 全国特等奖 | 解决大学生自己的问题（面试） |

**拿奖不靠技术多牛，靠的是：真实需求 + 合作方背书 + 你做得出。**

## 三、项目产品构想

### 极简产品轮廓

```
打开 → 选身份（大一/网络工程/会写代码）
     → 看到三个板块：

📋 我可以参加的比赛
   - 大创（报名时间：X月，截止还有30天）
   - 挑战杯AI专项赛（你的技术栈匹配：大模型应用、CV）
   - 蓝桥杯（你正在准备，比赛时间：X月）

🏭 黄石企业可以对接的项目
   - 湖北企点创：在找能做轻量级MES系统的大学生团队
   - 大唐融合：智能网联汽车方向有学生合作通道

💰 我可以申请的补贴
   - 大学生创业补贴5000元（毕业5年内，你在范围内）
   - 省级创业扶持2-10万
```

每条信息标注**原始出处链接**，用户可以一键跳转核实。

### 技术架构（一个界面走到头）

```
┌─────────────────────────────────┐
│         前端（一个网页）          │
│   用户输入：我是大一的，网络工程    │
│   选择关注：竞赛 / 企业项目 / 补贴  │
│         点一下搜索按钮             │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│         后端（调度层）            │
│                                 │
│  1. 拿用户画像 → 生成搜索关键词   │  ← 一个LLM
│  2. 调搜索API → 返回原始网页列表  │  ← 搜索API
│  3. 逐个读取网页内容             │  ← 网页抓取
│  4. 筛选 + 翻译成人话            │  ← 同一个LLM
│  5. 匹配用户画像打分             │  ← 同一个LLM
│  6. 标注来源 + 输出              │
│                                 │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│         外部服务（只需两个API）    │
│                                 │
│  🔍 搜索API（Brave/Bing/Tavily） │
│  🧠 LLM API（DeepSeek，便宜）    │
│                                 │
└─────────────────────────────────┘
```

### 产品壁垒（为什么不是随便一个Agent能做的）

1. **持续运行**：Agent是对话式的，不会主动推送。这个产品持续监控并主动通知。
2. **信息校验**：每条信息带原始链接，用户一键验证。Agent给的信息用户不放心。
3. **知识积累**：用户收藏的信息自动归档，形成个人化知识库。Agent聊完就没了。
4. **画像系统**：用户画像存一次，后续自动匹配。Agent每次都要重新告诉它你是谁。
