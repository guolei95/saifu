# 提示词：调研结果页面一键导出 PDF

> 复制以下内容发给另一个 AI，让它帮你实现。

---

## 项目背景

我有一个竞赛匹配网站（`saifu/`），匹配完成后结果直接渲染在页面上，排版已经很好了（竞赛卡片 + 建议 + 提示），可以直接给用户看。用户画像里有一个字段叫「**别名/称呼**」（用户自己填的，可能是真实姓名或微信号昵称）。

现在需要：在结果展示区加一个按钮，**一键打印为 PDF**，文件名用别名命名。

## 你的任务

在 `saifu/` 项目的结果展示区，加两个按钮：

| 按钮 | 功能 |
|------|------|
| 🖨 **打印 PDF** | 打开浏览器打印对话框，选「另存为 PDF」，文件名 `赛赋-调研报告-{别名}.pdf` |
| 💾 **保存 HTML** | 兜底方案，保存为 `.html` 文件（兼容手机端不能直接打印的情况） |

### 实现细节

#### 1. 获取别名

用户别名存储在页面上（可能是某个隐藏字段，或者是导入时解析出来的变量），你需要找到它。当前上报文本中别名字段示例：

```
别名/称呼：小明
```

在 `app.js` 中从解析出的用户数据里取 `user_data.alias`。

#### 2. 打印 PDF 按钮

```html
<button onclick="printToPDF()">🖨 打印 PDF</button>
```

```javascript
function printToPDF() {
  // 设置打印标题（影响 PDF 文件名，部分浏览器支持）
  document.title = `赛赋-调研报告-${alias}`;
  window.print();
}
```

`window.print()` 会自动打开浏览器打印对话框：
- **电脑端**：Chrome/Edge 的打印对话框里选「另存为 PDF」→ 文件名填 `赛赋-调研报告-小明`
- **手机端**：iOS Safari 支持直接打印为 PDF；Android Chrome 也支持

不需要任何后端代码，不需要 python-docx，浏览器自带。

#### 3. 保存 HTML 按钮（兜底）

```javascript
function saveAsHTML() {
```

#### 4. 文件名规则

- PDF 打印：`赛赋-调研报告-{别名}` （打印对话框里用户手工填）
- HTML 兜底：`赛赋-调研报告-{别名}.html` （自动下载，不用手工填）

「别名」是用户在问卷表单里填的称呼，方便小雷识别这是谁的报告。

## 约束

- 纯前端实现，不碰后端
- 打印/保存后样式和原页面一致
- 手机端也要能用（Safari 打印 / HTML 下载兜底）
- 不要破坏现有的匹配结果渲染逻辑
