## 一、技术架构（V1 CLI 版）

```
用户输入（学校/专业/年级/兴趣）
       ↓
[Python 脚本: match_competitions.py]
       ↓
Tavily 搜索 API ← 生成 6 条搜索关键词
       ↓
去重 → 20条网页结果
       ↓
LLM（Claude Opus 4.7，中转站）→ 两次调用
   ├─ 第1次：匹配「报名中」竞赛 Top5
   └─ 第2次：匹配「已截止」竞赛 Top5
       ↓
格式化输出（终端）
```

### API 依赖


| API               | 用途       | 费用          |
| ----------------- | -------- | ----------- |
| Tavily Search API | 搜索竞赛信息   | 免费额度1000次/月 |
| 中转站（api-siyu.top） | LLM匹配+提取 | 按量计费        |
| Claude Opus 4.7   | 模型       | —           |

## 二、输出字段设计


| 字段                    | 说明         | 示例                        |
| --------------------- | ---------- | ------------------------- |
| name                  | 竞赛全称       | 第三届中国高校计算机大赛-AIGC创新赛      |
| match_score           | 匹配度(0-100) | 90                        |
| match_reason          | 匹配理由       | AI+编程高度匹配...              |
| participation_type    | 参赛形式       | 仅个人 / 仅团队(3-5人) / 个人或团队均可 |
| registration_form     | 报名方式       | 个人/团队/学校推荐                |
| registration_url      | 报名链接       | https://...               |
| registration_deadline | 截止日期       | 2026-10-31                |
| suitable_majors       | 适合专业       | 计算机类、网络工程...              |
| cross_school_allowed  | 是否可跨校      | true/false                |
| is_free               | 是否免费       | true/false                |
| fee_amount            | 费用金额       | 免费 / 300元/人               |
| notes                 | 备注（含金量/难度） | A类竞赛，含金量高...              |
| source_url            | 来源URL      | https://...               |
| source_type           | 来源类型       | 官方 / 非官方聚合站               |


### 分组显示

- 🟢 **报名中** — 截止日期在当前日期之后，或尚未公布
- 🔴 **已截止** — 已过期但值得关注，供明年规划参考

## 四、运行方式

### Windows CMD

```bash

python --version

# 2. 确认文件存在
dir D:\.claude\match_competitions.py

# 3. 装依赖（首次）

# 4. 设密钥
```

## 六、设计原则

1. **不做好看没用的** — 解决真实痛点，不搞花架子
2. **做自己会用的** — 产品经理必须是用户
3. **先跑通再优化** — CLI 能跑就别急着做网页
4. **用户说需要才算** — 不做自嗨型产品
5. **差异化竞争** — 不和纯信息聚合网站比谁搜得多，而是比谁能「连接人」
