# 提示词：把调研结果页面直接保存为文件

> 复制以下内容发给另一个 AI，让它帮你实现。

---

## 项目背景

我有一个竞赛匹配网站（`saifu/`），匹配完成后结果直接渲染在页面上，排版已经很好了（竞赛卡片 + 建议 + 提示），可以直接给用户看。现在的问题是：**我需要把当前页面保存成一个文件发给用户，而不是截图。**

## 你的任务

在 `saifu/` 项目的结果展示区，加一个 **「💾 保存本页为文件」** 按钮。点击后把当前页面结果区的内容，保存成一个**自包含的 HTML 文件**，发给用户就能直接打开看。

### 核心要求：所见即所得

- **不要再重新排版，不要再生成 Word。**
- 直接把页面上已经渲染好的结果区域（`<div id="resultArea">` 里的所有内容），抓取下来，塞进一个独立的 HTML 文件
- 保存后的文件和页面上看到的**完全一样**

### 具体实现

#### 前端 JS（纯前端，不需要后端）

在 `saifu/index.html` 的结果展示区加一个按钮：

```html
<button onclick="savePageAsFile()">💾 保存本页为文件</button>
```

`savePageAsFile()` 函数要做的事：

1. **抓取内容**：获取 `document.getElementById('resultArea')` 的完整 innerHTML
2. **收集当前所有生效的 CSS**：
   - 遍历 `document.styleSheets`，把 `style.css` 里的规则全部内联
   - 遍历页面上所有 `<style>` 标签的内容
   - 确保保存后的文件不依赖任何外部 .css 文件
3. **构建独立 HTML**：
```javascript
const standaloneHTML = `<!DOCTYPE html>
<html lang="zh-CN">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>赛赋 SaiFu · 竞赛调研报告 — ${userName}</title>
<style>
/* 所有 CSS 内联在这里 */
${allCSS}
</style>
</head>
<body>
<div class="container">
${resultArea.innerHTML}
</div>
</body>
</html>`;
```
4. **触发下载**：
```javascript
const blob = new Blob([standaloneHTML], { type: 'text/html;charset=UTF-8' });
const url = URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url;
a.download = `赛赋-竞赛调研报告-${userName}.html`;
a.click();
URL.revokeObjectURL(url);
```

#### 额外要求

- **隐藏 JSON 数据段**：在文件末尾加一段不可见的 `<script type="application/json" id="sai-fu-data">`，放入用户画像 JSON。这样以后你还能把这个文件粘贴回本地 saifu 重新导入（配合导入调研功能）
- **文件命名**：`赛赋-竞赛调研报告-{用户名}.html`
- **手机兼容**：Blob + download 在大多数手机浏览器上也能触发下载（Safari 会弹「查看/保存」，微信内置浏览器可能需要引导「在浏览器中打开」）

### 不需要做的事

- ❌ 不需要后端 API
- ❌ 不需要 python-docx
- ❌ 不需要生成 Word/PDF
- ❌ 不需要重新排版

---

## 约束

- 纯前端实现，不碰后端
- 保存的文件打开后样式和原页面一致
- 手机端也要能触发保存
- 不要破坏现有的匹配结果渲染逻辑
