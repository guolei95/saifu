# 提示词：给 saifu 本地网站加「导入用户报告 + 自动调研」功能

> 复制以下内容发给另一个 AI，让它帮你实现。

---

## 项目背景

我有一个 Flask 后端 + 纯前端 HTML/JS 的竞赛匹配网站（`saifu/` 目录）。现在需要加一个新功能：**我收到用户微信发来的报告文本后，粘贴进网站，自动解析用户数据，然后对该用户进行个性化竞赛调研。**

## 你的任务

在现有的 `saifu/` 项目中新增以下功能：

### 1. 新增页面/组件：用户数据导入

在现有 `saifu/index.html` 上加一个新模块（可以是一个新的 tab 或者页面下方的区域），包含：

- **一个大文本框**：标题写「📋 粘贴用户报告」，placeholder 写「将用户微信发来的报告全文粘贴到这里...」
- **一个「🔍 开始调研」按钮**
- **一个结果展示区**：显示调研结果

### 2. 前端 JS：提取用户数据

用户粘贴的报告文本格式如下（示例）：

```
📋 赛赋·竞赛匹配报告
━━━━━━━━━━━━━━━━
别名/称呼：小明
学校：清华大学
专业：计算机科学与技术
年级：大三
兴趣领域：AI/大模型, Web开发, 数据分析
参赛目标：保研加分, 丰富简历
专业大类：工科
核心技能：Python编程, 机器学习, 深度学习
技能领域：AI应用/大模型, Web后端, 数据分析
常用工具：ChatGPT/Claude/Copilot等AI工具, Python/FastAPI/Flask, Git
每周可投入：每周10-15小时
空闲月份：7月, 8月
寒暑假可集中备赛：是
赛事级别：国家级
个人/团队：团队赛
有指导老师：有指导老师
接受跨校组队：可以跨校
比赛周期：1-3个月
比赛形式：线上提交
报名费：免费优先
语言：英文也行
需避免：纯商业计划书（无技术落地）, 纯笔试/死记硬背型
最高获奖：省级获奖
代表性项目：用Python写了个自动抢课脚本
有作品集/GitHub：有 | https://github.com/zhangsan/demo
最低接受：省级
理想目标：国家级一等奖
策略：含金量优先
学校有实验室：有
愿意加入校内团队：挑项目
需要匹配队友：需要
━━━━━━━━━━━━━━━━
生成时间：2026/05/11
```

**前端 JS 要做的事**：
1. 从报告文本中按行解析出所有字段（姓名、学校、专业、年级...等等，共约 30 个字段）
2. 把解析结果构建成 JSON 对象
3. 调用后端 API：`POST /api/import-and-research`，body 为 `{ "user_data": { 解析出的JSON对象 } }`
4. 把后端返回的调研结果展示在结果区

### 3. 后端 Python：新增 API 端点

在 `saifu/backend/main.py` 中新增一个路由：

```
POST /api/import-and-research
```

接收 body：
```json
{
  "user_data": {
    "name": "张三",
    "school": "清华大学",
    "major": "计算机科学与技术",
    "grade": "大三",
    "interests": ["AI/大模型", "Web开发"],
    "goals": ["保研加分", "丰富简历"],
    ... （所有字段）
  }
}
```

后端要做的事：
1. 把 user_data 存入本地 JSON 文件（`saifu/backend/data/users/` 目录下，文件名用 `{姓名}_{日期}.json`）
2. 调用现有的 AI 匹配逻辑（`services/ai_client.py` 里的 DeepSeek API），但 prompt 要换成一个**个性化调研 prompt**，不是原来的竞赛匹配 prompt
3. 返回调研结果

### 4. 调研 Prompt 设计

调研 prompt 应该让 AI 做以下分析：
- 根据用户画像，推荐 5-8 个最适合的竞赛
- 每个竞赛说明：名称、报名时间、为什么适合、需要准备什么
- 给出备赛建议（时间规划、技能补强、找队友策略）
- 如果用户有软肋（如指导老师缺失、技能短板），给出替代方案

### 5. 调研结果展示

前端展示调研结果时，按以下结构：
- 🏆 推荐竞赛列表（卡片式，和现有匹配结果样式一致）
- 📝 备赛建议（文本区）
- ⚠️ 风险提示（如果用户画像有短板）

---

## 约束

- 不破坏现有匹配流程
- 调研结果也要保存到本地 JSON（方便后续转 Word/PDF）
- 前端样式复用现有的 CSS 变量（`--accent`、`--card-bg` 等）
- 手机端也要能正常使用
