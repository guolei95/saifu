## 坑1：Windows CMD GBK 编码报错

- **现象**：`UnicodeEncodeError: 'gbk' codec can't encode character '❌'`
- **原因**：Windows CMD 默认 GBK 编码，不认 emoji（❌、✅、⭐ 等）
- **解决**：在文件头部加 `sys.stdout.reconfigure(encoding="utf-8", errors="replace")`
- **代码位置**：`match_competitions.py` 第 17-18 行
- **教训**：写 CLI 工具给 Windows 用户，要么不用 emoji，要么开头设 UTF-8

## 坑2：文件路径被转义

- **现象**：`D:\.claude\match_competitions.py` 变成 `D:\.claude\.claudematch_competitions.py`
- **原因**：反斜杠 `\m` 被 Bash 解释为转义字符
- **解决**：用 `D:/.claude/match_competitions.py` 正斜杠 + 正确路径
- **教训**：跨平台跑脚本，路径用正斜杠 `/`

## 坑3：环境变量跨命令不传递

- **现象**：`set KEY=value && python script.py` 中 Python 读不到环境变量
- **原因**：Bash 工具不是 cmd.exe，`set` 命令语义不同
- **尝试1**：`cmd /c "set ... && python ..."` — 失败，打开了交互式 shell
- **尝试2**：创建临时 .bat 文件执行 — 失败，同样打开交互式 shell
- **最终方案**：Bash 语法 `TAVILY_API_KEY="xxx" RELAY_API_KEY="xxx" python script.py`
- **教训**：Bash 工具运行在 bash/zsh 环境，不能用 cmd 语法

## 坑5：LLM 输出被截断（最折腾的坑）

- **现象**：`json.loads()` 报错，JSON 在 `"source_url": "https://` 处截断
- **原因**：中转站对单次请求有隐藏的输出长度限制，V1.1 用一次调用输出两组（open+closed）数据量太大
- **尝试1**：`max_tokens=4096` → 截断
- **尝试2**：`max_tokens=8192` → 仍然截断（中转站限制，非 token 限制）
- **最终方案**：拆成两次 LLM 调用
  - 第1次只查「报名中」的竞赛
  - 第2次只查「已截止」的竞赛
  - 每次返回独立 JSON 数组，数据量减半
- **附加防护**：`result_text.rfind("]")` 找到最后一个右括号，截掉后面不完整数据
- **副作用**：API 调用次数翻倍（耗时可接受，约 20 秒）
- **教训**：中转站/代理 API 可能有隐藏输出限制，大输出必须分片

## 坑6：同一个比赛在两个分组里重复出现

- **现象**：计算机设计大赛同时出现在 🟢报名中 和 🔴已截止 两个分组
- **原因**：两次 LLM 调用独立判断，不同搜索结果中截止日期信息不一致
- **当前状态**：未修复，视为可接受的小瑕疵
- **建议后续修复**：在 `print_results` 中对两个列表做 URL 去重

## 坑7：sohu.com 被误标为「官方」

- **现象**：来源是 `sohu.com`（搜狐自媒体），但 LLM 标了「🏛️ 官方」
- **原因**：prompt 里 source_type 规则不够细，只写了「.edu.cn/政府/学校域名」
- **当前状态**：已修 prompt，加了「搜狐/知乎/简书标非官方聚合站」
- **效果**：部分改善但非 100%，LLM 判断域名有时不准
- **建议后续修复**：在代码层做域名白名单，不在 prompt 里猜

## 坑10：中转站→DeepSeek 迁移

- **现象**：中转站存在两个无法修复的固有问题：
  1. 输出截断：JSON 在中间被拦腰切断（即使 max_tokens=8192）
  2. 中文引号损坏：U+201C → U+0022，导致 JSON 结构断裂
- **原因**：中转站对输出做了某种处理/限制，非标准 OpenAI 兼容行为
- **最终方案**：全面切换 DeepSeek 官方 API
  - 只需改 3 个环境变量（API Key、Base URL、Model），代码逻辑不变
  - DeepSeek 100% 成功率，零 JSON 解析错误
  - 费用标注质量大幅提升：能正确输出 300元/人、50元/人、49元/人
- **教训**：中转站/代理 API 的兼容性不可靠。直接用官方 API 是最稳妥的选择。看似省钱走中转站，实际踩坑成本远大于省下的钱。
