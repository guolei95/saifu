## 一、赛赋是什么

一句话：**AI驱动的竞赛全流程辅助平台，帮大学生从选赛到拿奖走完全程。**

### 核心功能

- 用户输入专业、兴趣、目标 → AI搜索最新竞赛信息 → 多层匹配打分 → 个性化推荐
- 不只搜比赛，还告诉你**这个比赛值不值得参加、有什么坑、怎么准备**
- 后续规划：匹配竞赛后分三个方向（大创/商赛/数模）全流程AI辅助

### 技术架构

```
用户浏览器 → Vercel前端(免费) → Cloudflare Tunnel(免费) → 本地FastAPI后端 → DeepSeek API + DuckDuckGo搜索
```

- **前端**：原生HTML/CSS/JS，响应式设计
- **后端**：Python FastAPI，6个模块（搜索/匹配/常识库/校验/AI客户端/主路由）
- **部署**：零月费，一个大一学生就能独立维护

| ------------- | ----------------- | ----------------------- | ------------------------- |

```
✅ 新架构：POST /api/match → 立即返回task_id → 每2s轮询GET → 完成返回
```

## 三、核心方法论：非代码人员如何用AI开发

### 3.1 你不是在写代码，你是在描述问题

和AI协作的核心不是「告诉AI怎么实现」，而是「告诉AI你想要什么结果」。

**错误的说法**：❌ 「用FastAPI写一个路由，接收POST请求，参数包括name和interests，返回JSON」

**正确的说法**：✅ 「用户在前端填了一个表单，包含专业和兴趣，按提交按钮后，后端收到请求，调用AI分析匹配，然后把匹配结果返回到页面展示出来。帮我把后端这个接口写好。」

**为什么**：AI比你懂技术实现，你描述场景和需求就好。

### 3.2 三窗口并行开发法（最值钱的方法）

这是第7天发现的最高效工作方式：

```
窗口1：搜索引擎功能 → 专注搜索模块
窗口2：知识库功能 → 专注数据层
窗口3：手机端适配 → 专注前端展示
```

**为什么有效**：

- 每个窗口的AI只专注一件事，上下文不会混乱
- 不需要在一段对话里切来切去
- 相当于三个不会沟通的程序员同时开发——各干各的，你来做集成

**前提条件**：

- 你的代码是模块化的（不然改一个地方三个窗口全崩）
- 每个模块有清晰的输入输出接口
- 你对整体架构心里有数

### 3.3 行动手册法（拆任务给AI的标准模板）

如果你要做一件复杂的事，不要只给AI一个模糊的需求。像这样结构化地交代：

```markdown
## 任务：把CLI工具改造成网页平台

### 约束
- 用户零门槛（不需要安装、不需要API Key）
- 后端持有API Key（前端不能出现）
- 架构可扩展

### 现有文件
- match_competitions.py（匹配逻辑，约2000行）

### 任务清单
1. 从现有代码提取匹配逻辑 → 写成独立函数
2. 搭建FastAPI后端（2个接口）
3. 写前端HTML页面（表单+结果展示）
4. 部署到Vercel（前端）+ 本地Tunnel穿透（后端）
```

AI拿到这种格式的指令，比拿到「帮我做个网站」准确10倍。

### 3.4 纠错循环法

这是AI开发的日常：

1. 你说需求 → AI写代码
2. 你测试 → 发现报错
3. 把报错信息原文贴给AI → AI修复
4. 再测试 → 可能还有新问题
5. 重复2-4，直到通过

**关键诀窍**：**把报错信息一字不改地贴给AI**。不要自己解读，不要「大概是什么什么问题」——直接贴原始error log。

### 3.5 架构第一，代码第二

在让AI写任何代码之前，先让他帮你梳理：

1. **这个系统有几个模块？** → 每个模块干什么
2. **模块之间怎么通信？** → 接口格式
3. **数据怎么流转？** → 从用户输入到最终输出的全路径

架构清楚了，后面写代码就是填空题。架构不清楚，写出来的代码就是一坨乱麻，改一处崩三处。

## 八、给同样想用AI开发的人的建议

### 8.1 心态篇

1. **不要等「学会了再开始」**：我第一天连FastAPI是什么都不知道，AI会告诉你。
2. **接受「改10次才成功」**：AI第一版代码很少能跑通，纠错循环是正常流程。
3. **你是产品经理，AI是程序员**：你的价值在设计、判断、决策，不在写代码。
4. **便宜模型够用了**：DeepSeek这种级别的模型对MVP绰绰有余，不用上最贵的。

### 8.2 流程篇

```
你的角色：需求 → 测试 → 判断 → 决策
AI的角色：搜索 → 分析 → 编码 → 修复 → 文档
```

**标准开发循环**：

1. 说需求（描述你想要什么，不要描述怎么实现）
2. AI写代码
3. 你运行测试

### 8.3 省钱篇


| 策略                          | 说明                   |
| --------------------------- | -------------------- |
| 用DeepSeek而不是Claude          | 便宜5倍，质量够用            |
| 搜索用DuckDuckGo免费API          | 代替Tavily等付费搜索        |
| 部署用Vercel+Cloudflare Tunnel | 完全免费，学生项目够用          |
| 不要过早买服务器                    | 先用免费方案验证，有人用了再升级     |
| 知识库硬编码                      | 数据量小的时候，JSON文件比数据库省事 |


### 8.4 避坑篇

1. **直接走官方API，别用中转站**：看似省钱，踩坑成本远大于省下的钱。
2. **先做CLI验证，再做网页**：逻辑跑通了再投入前端，不然UI做完了发现核心逻辑不对。
3. **免费隧道不适合长连接**：如果你的接口需要超过30秒响应，必须用异步轮询模式。
4. **API Key放后端环境变量**：前端代码里绝对不能出现，GitHub公开仓库会泄露。
