### 问卷核心问题设计（建议）
1. 你参加过大学生竞赛吗？（是/否）
2. 如果没参加，主要原因是什么？（不知道有什么比赛 / 不知道怎么准备 / 找不到队友 / 怕浪费时间和报名费 / 觉得自己水平不够）
3. 你通过什么渠道了解竞赛信息？（班级群/辅导员/学长学姐/公众号/自行搜索）
4. 你最需要哪种竞赛帮助？（选赛推荐 / 方案写作 / 备赛规划 / 找队友 / 获取往届资料）
5. 你愿意为竞赛辅助工具付费吗？（愿意 / 看价格 / 不愿意）
6. 你的学校、专业、年级？（基础画像）

### 访谈问题建议
不做问卷式问答，而是引导对方讲故事：
- 「你上次想参加比赛是什么时候？后来怎么样了？」
- 「你准备比赛的时候，最难的是什么？」
- 「如果有人告诉你xx比赛很适合你现在的情况，你会去看吗？」

### 信息沉淀后的输出结构
```
01-调研/050X-市场调研总报告.md
├── 一、调研方法（问卷N份 + 访谈N人）
├── 二、用户画像（双非大学生的典型竞赛困境）

└── 六、差异化定位结论
```

## ④ 根据项目方案做计划书框架

### 前置依赖
- ① 的调研数据 → 撑起「市场分析」「用户需求」两章
- ② 的完整方案 → 撑起「产品介绍」「技术方案」两章
- ③ 的商业化 → 撑起「商业模式」「财务预测」两章

### 商业计划书标准结构（适配互联网+/大创）

| 章节 | 内容 | 依赖 |
|------|------|------|
| 1. 执行摘要 | 项目一句话 + 核心亮点 | ①②③全做完后提炼 |
| 2. 项目背景 | 为什么做这个？痛点+数据 | ① |
| 3. 市场分析 | 市场规模、竞争格局、差异化定位 | ① |
| 4. 产品介绍 | 双赛道架构、核心功能、用户流程 | ② |
| 5. 技术方案 | 技术架构、提示词体系、数据流向 | ② |
| 6. 商业模式 | 盈利模式、定价、市场策略 | ③ |
| 7. 团队介绍 | 核心成员、分工、顾问 | ⑤ |
| 8. 发展规划 | 产品路线图、里程碑 | ② |
| 9. 财务预测 | 收入/成本/利润预测 | ③ |
| 10. 风险与应对 | 主要风险及应对策略 | 全局 |
| 附录 | 调研数据、软著证明、用户反馈 | ① |

### 计划书框架不应该是「空壳」
每一章应该至少有：
- 该章的论述逻辑（先说什么后说什么）
- 需要填的数据/素材（标注来源）
- 至少挑2-3章写出完整初稿作为样板

## ⑤ 信息总结后找指导老师和队友

### 前置条件
前四项做完后，你手上有：
- 一份有数据的市场调研报告
- 一份完整的项目方案（含提示词）
- 一份清晰的商业计划书框架
- 一个能跑通的CLI原型

这些材料足够让一个老师/同学在5分钟内理解你的项目。

### 找指导老师

**目标画像**：
- 专业课老师中比较年轻、对AI感兴趣、指导过学生竞赛的
- 或者辅导员中负责竞赛/创新创业的老师
- 优先找「知道你是谁」的老师（上过课、聊过天）

**怎么说**（核心话术）：
> 「老师，我是网络工程xx班的学生，在做一个大创/互联网+项目——AI驱动的竞赛辅助平台，特别针对我们双非院校学生的信息差问题。产品原型已经跑通了，调研也做了，想请您帮忙指导一下，主要是在商业模式和比赛方向上给点建议。」

**准备材料**：
- 一张A4纸的项目简介（不是60页计划书）
- 手机展示CLI跑通的效果（或网页截图）

### 找队友

**需要的人**：
| 角色 | 需要能力 | 找谁 |
|------|---------|------|
| **商业/文案** | 商业计划书写作、PPT设计、路演表达 | 经管学院同学 |
| **设计/前端** | UI设计、网页制作 | 计算机同学或设计专业 |

**怎么说**（核心话术）：
> 「我在做一个竞赛项目，AI驱动的竞赛助手，产品已经跑通了。现在缺一个写商业计划书/做PPT的搭档，如果你加入，我们一起冲省级以上奖项——这对你保研加分也很有用。」

**怎么找**：
- 班级群直接问（附上项目简介）
- 找辅导员推荐（辅导员知道谁在找比赛机会）
- 食堂/图书馆贴一张A4纸（手写更真诚）
