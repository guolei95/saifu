## 四、合并建议

### 每条竞赛统一结构
```json
{
  "id": 从1开始,
  "name": "竞赛全称",
  "official_url": "来自DOCX",
  "category": "来自DOCX或推断",
  "timing": "报名时间（来自XLSX）",
  "format": "赛制说明（来自XLSX）",
  "work_types": "作品类别（来自XLSX）",
  "requirements": "参赛要求（来自XLSX）",
  "side_events": "配套赛事（来自XLSX）"
}
```

### 三步合并法
1. **自动匹配**：用名称关键词匹配两个文件的条目
2. **人工抽查**：匹配失败的条目输出清单，人工确认
3. **整合已有**：赛赋已有的9条口语化字段保留，与新结构化字段共存

## 八、补充建议

### 优先级1：补84条竞赛数据（最大缺口）
将 DOCX(URL) + XLSX(详情) 合并为统一 JSON，塞进赛赋知识库。skill 也能引用这份数据做推荐。

### 优先级2：去重案例库
`REAL_CASES` 和 `05-cases.md` 是同一批案例，建议保留skill的详细版，赛赋只用引用/同步。

### 优先级3：好处+误区合并
Skill详细版 + 赛赋精简版 → 统一维护一处。赛赋代码需要的短文本可以从详细版抽取摘要。

### 可选：赛赋独有能力反向输出给skill
`classify_competition` 的分类逻辑、`enrich_with_facts` 的校验方法，skill在推荐竞赛时也能用上。
