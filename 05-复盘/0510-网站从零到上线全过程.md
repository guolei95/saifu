# 赛赋网站：从零到上线，让别人能打开的全过程

> 一个不会写代码的大一学生，7天把网站做到「发链接给别人就能用」。

---

## 一、整体路线图

```
第1步：CLI原型验证      →  Python命令行工具，确认核心逻辑能跑通
第2步：拆成后端+前端     →  FastAPI后端(处理数据) + HTML页面(展示界面)
第3步：前端部署到Vercel  →  得到一个永久网址，别人能打开看到页面
第4步：后端用Tunnel穿透  →  本地电脑的后端，通过Cloudflare生成公网URL
第5步：前后端对接        →  前端页面调用后端API，完整功能跑通
第6步：架构优化          →  解决Tunnel不稳定问题（提交+轮询模式）
第7步：Vercel被墙→寻找国内方案  →  尝试Cloudflare Pages，同样被墙
第8步：阿里云买域名      →  saifu.asia（¥8/年），DNS指向Cloudflare Pages
```

## 二、每一步具体做了什么

### 第1步：CLI原型（5月3日）

**产出**：`match_competitions.py`（约2000行Python）

**做了什么**：
- 用户在终端里填信息 → Python调用DuckDuckGo搜竞赛 → DeepSeek AI匹配打分 → 终端打印结果
- 验证了核心逻辑能跑通：搜索有效、AI匹配有意义、输出格式合理

**为什么先做CLI**：不要一上来就写网页。先用最简单的方式验证想法对不对，逻辑跑通了再考虑界面。

---

### 第2步：拆成后端+前端（5月7-8日）

**做了什么**：把2000行的单文件拆成模块化的Web架构。

**后端**（`saifu/backend/`）— 6个Python模块：
```
main.py          ← FastAPI应用入口，定义API接口（用户提交画像、查看结果）
config.py        ← 配置管理，DeepSeek API Key存这里
engine.py        ← 匹配引擎，8步流程（搜索→匹配→校验→自审→排序）
search.py        ← DuckDuckGo搜索，生成搜索词+执行搜索
ai_client.py     ← DeepSeek API调用封装
knowledge_base.py ← 常识库（9个竞赛数据+校验函数）
validation.py    ← 交叉验证+AI自审查
```

**前端**（`saifu/frontend/` 或 `saifu/`）— 3个文件：
```
index.html  ← 页面结构（表单+结果卡片+加载动画）
style.css   ← 页面样式（响应式，手机电脑都能看）
app.js      ← 前端逻辑（收集表单数据→调后端API→渲染结果）
```

**关键设计决策**：
- 前端用原生HTML/CSS/JS，不上React/Vue（功能简单不需要框架，AI对原生代码生成质量更高）
- API Key只放在后端环境变量里，前端代码里绝对不能出现（不然GitHub公开仓库会泄露）

### 第3步：前端部署到Vercel（得到永久网址）

**做了什么**：把前端HTML/CSS/JS部署到Vercel，得到一个别人能打开的网址。

**具体操作**：
1. 去 [vercel.com](https://vercel.com) 用GitHub账号注册
2. 在Vercel里Import GitHub仓库
3. 设置Root Directory为 `saifu`（前端文件所在的文件夹）
4. 部署后自动获得一个域名：`https://saifu-75e4.vercel.app`

**Vercel配置**（`saifu/vercel.json`）：
```json
{
  "version": 2,
  "builds": [{ "src": "**", "use": "@vercel/static" }]
}
```

**为什么选Vercel**：
- 完全免费
- 有香港节点，国内访问速度快
- 推送GitHub代码自动部署（改一行push一下，30秒生效）

---

### 第4步：后端用Cloudflare Tunnel穿透（让本地后端有公网URL）

**问题**：后端跑在你自己电脑上（localhost:8000），只有你能访问。怎么让别人也能调？

**解决方案**：Cloudflare Tunnel —— 免费工具，把你电脑上的端口映射成一个公网URL。

**具体操作**：
1. 下载 `cloudflared.exe`（Cloudflare Tunnel客户端）放到 `C:\Users\guolei\`
2. 先启动后端：
   ```bash
   cd saifu/backend
   python -m uvicorn main:app --host 0.0.0.0 --port 8000
   ```
3. 再启动Tunnel：
   ```bash
   cloudflared.exe tunnel --url http://localhost:8000
   ```
4. 日志中会出现类似 `https://xxx.trycloudflare.com` 的URL —— 这就是你的后端公网地址

**注意**：
- Tunnel URL每次重启电脑后会变（新的随机域名）
- 免费版无SLA，偶尔会断连

---

### 第5步：前后端对接

**做了什么**：让前端页面能调到后端API。

**具体操作**：
1. 在 `saifu/js/app.js` 里设置API地址：
   ```javascript
   const API_BASE_URL = "https://xxx.trycloudflare.com";  // 你的Tunnel URL
   ```
2. 前端提交表单后，JS调用 `API_BASE_URL + "/api/match"` 发送用户画像
3. 后端处理完返回结果，JS渲染成卡片展示

**部署流程**（改API地址后）：
```bash
git add saifu/js/app.js
git commit -m "fix: 更新Tunnel URL"
git push
```
Vercel自动检测到push，30秒后线上生效。

**最终用户访问链路**：
```
别人打开 https://saifu-75e4.vercel.app
    → 看到表单页面（来自Vercel）
    → 填写信息点提交
    → 前端JS调 https://xxx.trycloudflare.com/api/match（Tunnel穿透到你电脑）
    → 你电脑上的FastAPI处理请求 → 调用DeepSeek+DuckDuckGo
    → 返回匹配结果 → 页面展示卡片
```

### 第6步：架构优化——解决Tunnel不稳定（最关键的技术改造）

**遇到的问题**：匹配流程需要60-180秒（搜索+多轮AI调用），但Cloudflare免费Tunnel对超过60秒的长连接极不稳定，经常断开。

**现象**：用户等了两分钟，结果页面报「无法连接到服务器」。

**改造方案**：从「单次长请求」改成「提交+轮询」模式。

```
❌ 旧架构（会断）：
   浏览器 ──POST一次请求──→ 等60~180秒 ──→ 返回结果
   中途Tunnel断开 → 全丢 → 用户重试 → 再次断开

✅ 新架构（不断）：
   浏览器 ──POST提交画像──→ 立即返回task_id（不到1秒）
   浏览器 ──每2秒问一次：好了吗？──→ processing / done
   单次轮询失败自动重试，不受Tunnel波动影响
```

**改动了3个文件**：
| 文件 | 改动 |
|------|------|
| `backend/main.py` | POST立即返回task_id + 后台异步处理 + GET轮询接口 |
| `backend/services/ai_client.py` | 加 `timeout=120.0` 防卡死 |
| `saifu/js/app.js` | 单次fetch → 循环轮询，轮询失败自动重试 |

### 第7步：面向中国用户——Vercel被墙，寻找国内方案

**问题**：Vercel的默认域名 `saifu-75e4.vercel.app` 在国内被墙。你在国外能打开，但国内同学打不开。

**为什么会这样**：Vercel用的是 `*.vercel.app` 泛域名，这个域名在中国大陆被DNS污染/屏蔽。免费境外托管服务普遍有此问题。

**尝试的替代方案**：

| 平台 | 地址 | 国内访问 |
|------|------|---------|
| Vercel | `saifu-75e4.vercel.app` | ❌ 被墙 |
| Cloudflare Pages | `saifu.3359717058.workers.dev` | ❌ 也不通 |

**结论**：任何 `*.vercel.app` 或 `*.workers.dev` 这种境外免费域名，国内大概率都访问不了。要让国内用户能打开，必须走**自己的域名**。

---

### 第8步：阿里云买域名 + DNS指向

**做了什么**：在阿里云万网买了一个专属域名，让国内用户通过这个域名访问。

**具体操作**：

**1）购买域名**：
- 平台：阿里云万网（[wanwang.aliyun.com](https://wanwang.aliyun.com)）
- 域名：`saifu.asia`
- 费用：**¥8/年**（`.asia` 后缀首年特价）
- 购买后需实名认证（上传身份证，几分钟审核通过）
- 域名进入「注册局审核中」状态（约8小时）

**2）配置DNS解析**：
- 进入阿里云「域名解析」控制台
- 添加 CNAME 记录，把域名指向你托管前端的地方：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| CNAME | @ | `saifu.3359717058.workers.dev`（Cloudflare Pages地址） |

或者如果指回 Vercel：
| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| CNAME | @ | `cname.vercel-dns.com` |

**为什么是CNAME不是A记录**：A记录指向固定IP，但Vercel/Cloudflare Pages的IP会变。CNAME指向他们的域名，他们自己管IP变化。

**3）在托管平台绑定自定义域名**：
- Vercel：项目设置 → Domains → 添加 `saifu.asia`
- Cloudflare Pages：项目设置 → Custom domains → 添加 `saifu.asia`

**4）等DNS生效**：
- 阿里云DNS修改后，国内生效约10分钟-2小时
- 用 `ping saifu.asia` 验证是否指向了正确的地方

### 🔮 终极方案：如果自定义域名+境外托管仍然被墙

**可能的问题**：即使绑了自己的域名，如果DNS解析到的IP是Cloudflare Pages/Vercel的境外IP，国内仍可能被限速或间歇性阻断。

**真正的国内可用方案**：阿里云OSS静态网站托管 + 阿里云CDN + ICP备案

```
用户打开 https://saifu.asia
    → 阿里云CDN（国内节点，速度飞快）
    → 阿里云OSS（存储HTML/CSS/JS文件）
    → 前端JS调后端API
    → 后端部署到阿里云ECS（或继续保持Tunnel）
```

**代价**：
- OSS存储：¥0.12/GB/月（学生量几乎为0）
- CDN流量：¥0.24/GB（小网站前几个月可能就几毛钱）
- ICP备案：免费，但需要提交资料+等待约20个工作日
- ECS服务器：¥34/月起（如果不想用本地Tunnel）

**是否现在做**：建议先等 `saifu.asia` 域名生效 + Cloudflare Pages绑定测试。如果国内确实能访问，就先不搞备案。如果不行，再做OSS+CDN方案。
