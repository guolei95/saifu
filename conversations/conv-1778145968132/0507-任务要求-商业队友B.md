## 第8章：财务分析 ⭐ 依赖第6章定价

> 建议字数：800-1000字 | 预估：3-4h | 逻辑自洽即可，不要编假数据

### 要写什么

| 小节 | 请写这些内容 |
|:--|:--|

| 8.4 资金来源 | 创业补贴5000 + 省级扶持2-10万 + 自筹 |
| 8.5 盈亏平衡分析 | 需要多少付费用户覆盖成本 |
| 8.6 财务风险控制 | API成本管控 + 付费转化率底线 |

### 用这些素材

| 文件 | 用途 |
|:--|:--|
| `02-方案/0506-方案框架-赛赋.md` | 第8章框架，有资金来源和风险控制 |
| `02-方案/0506-执行规划.md` | 大模型API成本方向 |
| `01-调研/0503-黄石高校与企业调研报告.md` | 创业补贴5000元等政策数据 |

### 需要你查的具体数据

- [ ] **大模型API单价**：查DeepSeek官方API定价，算单次调用成本
- [ ] **服务器费用**：查阿里云/腾讯云学生优惠价
- [ ] **软著申请费用**：查代理机构报价（通常1000-3000元）
- [ ] **收入预测表**：用Excel建三档模型（保守/中性/乐观），截图放入

### 怎么写

- **不做假账，数据宁可保守不夸大**
- 成本大头是API调用，算清楚：每次选赛推荐消耗XX token × 日活用户数 = 日成本
- 盈亏平衡点：保守估计，**100个付费用户（×19.9元/月）即可覆盖初期成本**
- 不要写「第一年营收500万」，写一个**保守但自洽**的模型
- 收入预测做Excel表截图放入（不要让设计队友帮你做表）

### 怎么写

```
风险描述（1句）→ 可能性（高/中/低）→ 影响程度 → 应对措施（2-3句）
```
