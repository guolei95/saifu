# 赛赋部署架构升级：从本地隧道到全云端

> 2026-05-11，一句话总结：别在本地跑服务。

---

## 旧架构（踩过的坑）

```
用户 → Vercel（前端）→ Tunnel（临时域名）→ 你电脑（后端:8000）
```

- 后端跑在你电脑上，关机就挂
- 每次重启拿新隧道 URL，手动改代码再 push
- 5/9-5/11 三天困在"断→重启→换URL→又断"的循环里
- 日志里 tunnel/URL/重启 出现 20+ 次

## 新架构（现在的方案）

```
用户 → GitHub Pages（前端）→ Render（后端，新加坡）
                                        ↓
                                  DeepSeek API
```

- 前端：GitHub Pages，push 自动部署，免费
- 后端：Render Web Service，固定域名，免费（750h/月）
- 全云端，你关机也不影响

## 给后来者的经验

1. **能放云端就放云端**。Render/GitHub Pages 免费够用，别在本地跑
2. **不要依赖临时隧道**。Cloudflare Tunnel / localhost.run 都不可靠
3. **前端用 GitHub Pages**。比 Vercel 更稳，push 自动部署
4. **API Key 别写进代码**。放环境变量，Git 历史里有就换掉
5. **部署先做对，再做多**。花在救火上的时间够部署 10 次了

## 迁移步骤（备忘）

```
1. 后端加上 Procfile + requirements.txt
2. Render 新建 Web Service，连 GitHub 仓库
3. 环境变量填 DEEPSEEK_API_KEY
4. 前端 app.js 把 API_BASE_URL 改成 Render 地址
5. GitHub Pages 选 gh-pages 分支部署
6. 完事
```
