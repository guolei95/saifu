## V3 架构说明

系统根据用户入口自动归类到三个方向，内部调度对应提示词：

```
                            用户入口
                               │
                    ┌──────────┴──────────┐
                    │                     │
              Path A: 知道比赛       Path B: 不知道比赛
              填赛事+专业年级          填专业+特长+兴趣+目标
                    │                     │
                    ▼                     ▼
              直接分类到方向         Python 多维度打分推荐
                                            │
                                    用户从推荐列表选择
                                            │
                    └──────────┬──────────┘
                               ▼
                         方向自动归类
                    ┌─────────┼─────────┐
                    ▼         ▼         ▼
              🏗️ 大创方向  💼 商赛方向  📊 数模方向
              (方案+答辩)  (商业分析)   (建模/竞技)
                    │         │         │
                    └────┬────┘    ┌────┴────┐
                         │         │         │
                    通用层        数模路线   知识竞技变体
                    调研/方案/    建模+论文   学习+刷题+模考
                    PPT路演
                         │         │         │
                         └────┬────┴────┬────┘
                              ▼         ▼
                         赛后沉淀（三方向通用）
```

## 使用方式

每个提示词文件包含完整的提示词模板（角色 + 任务 + 信息收集 + 执行 + 约束），复制到 AI 即可运行。系统内部自动调度，用户无需指定用哪个提示词。

## 原则

1. **AI 出初稿，人做判断** — AI 给你参考，最终让人拍板
2. **分步推进** — 写完一部分确认了再继续
3. **交叉验证** — 关键事实用至少两个来源确认
4. **数据铁律** — 不编造任何数据、案例、引用，无出处标注「⚠️ 待确认」
