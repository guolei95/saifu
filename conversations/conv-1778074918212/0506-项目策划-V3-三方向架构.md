# 项目流程优化 v3 — 三方向架构 + 通用层

> 基于 `0505-项目整体策划与想法.md` 优化。核心变化：
>
> 1. 主架构从「方案赛道/竞技赛道」重构为「三方向 + 通用层」
> 2. 入口分 A/B 两条路径（知道比赛 / 不知道比赛），统一走到方向分类
> 3. 系统内部自动调度提示词，用户无需知道「工具分工」
> 4. 每个模块统一五段式：信息收集 → 参考分析(可选) → 核心执行 → 输出示例 → 约束

## 二、三方向覆盖范围

### 🏗️ 大创方向

覆盖所有「方案写作 + 答辩展示」类比赛：
大创、互联网+、挑战杯、三创赛、创青春、中国创翼、各类省市级创新创业大赛

核心特征：需要产出完整项目方案/申报书，最终路演答辩

### 💼 商赛方向

覆盖所有「商业分析 + 案例研究」类比赛：
欧莱雅、宝洁、联合利华、贝恩、罗兰贝格、奥纬、德勤数字化、各类企业案例分析赛

核心特征：需要市场分析、商业逻辑、策略方案，注重立意和创新

### 📊 数模方向（主）+ 🎯 知识竞技变体

- **数模路线**：国赛、美赛、MathorCup、深圳杯、电工杯、亚太杯等。核心特征：限时高压、建模+编程+论文三位一体
- **知识竞技变体**（不复用数模的建模/论文模块，但复用备赛/学习/刷题/模考模块）：ACM/蓝桥杯/天梯赛（编程）、大学生英语竞赛（考试）、数学竞赛（考试）、电子设计/机械创新（工科）

> 知识竞技不单独列方向——它的流程 = 数模的「备赛→学习→刷题→模考」前半段，去掉建模和论文部分。用户入口归到数模方向后，系统根据竞赛类型自动裁切流程。

## 三、入口设计 — A/B 双路径

```
用户打开系统
    │
    ├── Path A：「我知道要参加什么比赛」
    │   输入：赛事名称 + 赛道/组别 + 专业年级 + 特长兴趣 + 目标
    │   → 系统根据赛事名称自动归类到三方向之一
    │   → 跳过选赛推荐，直接进入方向流程
    │
    └── Path B：「我不知道能参加什么」
        输入：专业年级 + 特长 + 兴趣 + 目标
        → 系统跑 Python 多维度打分推荐（match_score 0-100 +
           recommend_index 1-5星 + match_reason 三段式）
        → 用户从 Top 5 推荐列表中选一个
        → 系统根据选中赛事自动归类到三方向之一
        → 进入方向流程
```

### Path B 打分维度（已有 Python 代码实现）


| 维度       | 权重  | 说明                 |
| -------- | --- | ------------------ |
| 专业匹配度    | 30分 | 竞赛内容与专业的对口程度       |
| 兴趣/目标匹配度 | 30分 | 是否匹配用户兴趣方向和参赛目标    |
| 年级/时间合适度 | 20分 | 年级是否在推荐参赛范围、时间是否冲突 |
| 可操作性     | 20分 | 时间投入、队友需求、报名门槛等    |


输出时映射为 `match_score`（0-100 分）+ `recommend_index`（1-5 星）+ `match_reason`（三段式）。

### 方向归类规则


| 竞赛关键词                      | 归类方向                  |
| -------------------------- | --------------------- |
| 大创、互联网+、挑战杯、三创赛、创青春、创业     | 🏗️ 大创方向              |
| 宝洁、欧莱雅、联合利华、贝恩、商赛、案例       | 💼 商赛方向               |
| 数模、建模、美赛、国赛、MathorCup      | 📊 数模方向（建模路线）         |
| 蓝桥杯、ACM、天梯赛、英语竞赛、数学竞赛、电子设计 | 📊 数模方向（知识竞技变体）       |
| 演讲、辩论、模联、职业规划              | 🏗️ 大创方向（复用 PPT 路演模块） |

### 阶段1：调研

> 对应 `prompts/02-调研/`（往届作品拆解 + 市场行业调研 + 调研报告生成）

```
入口 ─┬─ ① 往届作品拆解（用户有参考材料才执行）──┐
      │                                            │
      └─ ② 市场行业调研 ──────────────────────────┤
                                                   ▼
                                             ③ 调研报告生成
```

#### ① 往届作品拆解（可选）

**前置条件：** 用户提供了往届获奖作品（至少 2 份）

**执行：**

- 逐份拆结构（章节划分、篇幅占比、论证逻辑）
- 横向对比找共性模式（至少 5 个）
- 提取差异亮点（某份作品独有的高分点）

**兜底路径（无往届作品）：**

- 基于公开信息做通用分析
- 建议用户去学校官网/学长学姐处收集
- 标注「⚠️ 样本不足，结论仅供参考」

**输出要点：**

- 获奖清单：必须有什么、最好有什么、绝对不能有什么
- 3 个可直接借鉴的具体写法（带出处）

#### ② 市场/行业调研

**信息收集：** 调研方向 + 当前阶段

**系统自动调度搜索工具**（用户无需指定用什么工具）

**输出结构：**

1. 关键事实（5-8 条，附数据来源）
2. 核心矛盾（2-3 个）
3. 值得深挖的线索（3-5 条）
4. 数据清单（来源 + 年份）

#### ③ 调研报告生成

**输出示例：**

```
**核心发现 1：大学生心理咨询供需严重失衡**
教育部 2024 年数据显示，全国高校心理咨询师与学生比为 1:4000，
远低于国际标准 1:1500。某 985 高校调研显示，68% 学生表示
「想咨询但排不上号」...
```

**约束：**

- 不编造数据，无来源标注 ⚠️
- 不超 3000 字
- 不给空泛建议（如「多做调研」）

### 阶段2：方案设计

> 对应 `prompts/03-方案设计/`（框架搭建 + 初稿生成 + 分章节撰写 + AI 评委审查）

```
搭框架 → 写初稿(分步/通用) → AI 评审 → 审改迭代(循环)
```

#### 2.1 搭框架

**信息收集：** 比赛 + 方向 + 提交要求 + 已有调研产出 + 团队情况 +（可选）往届获奖方案参考

**参考分析（有参考才执行）：** 拆解获奖方案章节结构和篇幅分配

**输出要点：**

- 每章：标题 + 核心问题 + 3-5 个要点 + 篇幅占比 + 建议负责人
- 章节间逻辑链（前一章结论如何支撑下一章）
- 一句话方案逻辑线：「因为 A → 所以 B → 我们做 C → 能达到 D」

**约束：**

- 只出骨架不写正文
- 篇幅占比 = 100%
- 团队未组队则跳过负责人分配

#### 2.2 写初稿（通用版）

一次产出完整方案骨架 + 要点大纲。适合快速拿到可讨论版本。

**约束：**

- 3000-5000 字框架
- 不编造市场/用户数据
- 创新点必须具体

#### 2.3 写初稿（分步版）⭐ 推荐

**信息收集：** 要写哪一章 + 完整方案框架 + 前序章节状态 +（可选）额外素材/参考章节

**执行顺序：**

1. 回顾框架中本章核心问题
2. 检查前序章节结论，自然承接
3. 按要点逐个展开为完整段落
4. 自查：问题是否被回答？有无矛盾？数据是否有来源？

**约束：**

- 一次只写一章，用户确认后再下一章
- 每段第一句必须是核心结论
- 无数据处标注 📊 需补充
- 无过渡废话（「众所周知」「随着科技的发展」）

#### 2.4 AI 评委审查

**信息收集：** 比赛 + 方案全文 +（可选）答辩时间限制/评审标准

**评审维度（1-10 分）：**
选题价值 | 逻辑严密性 | 创新性 | 可行性 | 完成度 | 表达质量

**输出结构：**

- 整体第一印象
- 逐项打分 + 理由
- 3 个致命问题
- 3 个加分机会
- 3 个模拟尖锐问题 + 建议回答方向

**约束：**

- 不要当好人，该给 3 分给 3 分
- 问题必须具体
- 质量差就直接说「推翻重来」

#### 2.5 审改迭代循环

```
第 1 轮：结构性问题 → 第 2 轮：内容问题 → 第 3 轮：表达问题
    ↓                      ↓                      ↓
  逻辑不通               数据不够               啰嗦不专业
  章节缺失               论证薄弱               格式不统一
  重点不突出             创新点不清晰
```

**循环机制：**

- 每轮换一个审的角度，不重复审同样的东西
- 至少跑 3 轮

**退出条件：**

- ✅ 用户说「可以了」
- ✅ AI 评审各项 ≥ 7 分
- ✅ 最多迭代 5 轮后强制提示人工确认

### 阶段3：PPT 与路演

> 对应 `prompts/05-PPT与路演/`（PPT 结构优化 + 模拟答辩 + 演讲稿打磨）

```
PPT 文案结构 → 模拟答辩(循环练习) → 演讲稿打磨
```

#### 3.1 PPT 结构优化

**信息收集：** 方案全文 + 路演时长/页数限制 +（可选）往届获奖 PPT

**输出（不包含视觉设计）：**

- 总页数（按每页 30-45 秒估算）
- 整体逻辑线（一句话串联）
- 每页：核心信息 + 建议内容 + 讲述时长
- 3 页「决胜页」标注
- 多余页检查

**约束：**

- 每页一个核心信息
- 前 3 页必须抓住评委
- 最后一页不用「谢谢」
- 文字不超方案全文 10%

#### 3.2 模拟答辩

**流程：**

```
AI 提问（最尖锐的）→ 用户回答 → AI 点评 → 下一问
                                            ↓
                                   所有问题结束 → 总评
```

**点评结构：**

- 评委真实反应（满意/存疑/不买账）
- 哪里答得好
- 哪里没答到点
- 建议回答方向

**约束：**

- 一次只问一个问题
- 基于方案内容提问，不泛问
- 回答好就直说好，不硬挑毛病

#### 3.3 演讲稿打磨

**处理流程：**

1. 按 160 字/分钟估算当前时长
2. 逐段口语化改写（「基于上述分析」→「所以你看」）
3. 长句拆短句（≤ 25 字）
4. 标注语气重音（**加粗**）和停顿（[停 1 秒]）
5. 输出控时检查表

**约束：**

- 不改变内容和观点
- 专业术语保留 + 一句大白话解释
- 超时 > 120% 时先列出建议删减项，不自行删减

## 六、📊 数模方向流程

```
选赛 → 备赛规划 → 学习+刷题 → 模拟实战 → (建模路线+论文) → 赛后沉淀
```

> 数模方向含两条子路线，系统根据竞赛自动选择：

### 建模路线（国赛/美赛等）

> 完整流程，包含建模 + 编程 + 论文

**72 小时时间线：**


| 时间块          | 任务              | 产出         |
| ------------ | --------------- | ---------- |
| 读题/选题（2-3h）  | 拆题、匹配模型、分工      | 选定题目 + 分工表 |
| 分析题目（3-4h）   | 子问题拆解、数据梳理      | 解题框架       |
| 建模编程（24-30h） | 模型选择、代码框架、debug | 模型 + 代码    |
| 写论文（16-20h）  | 按标准格式分章节写       | 论文初稿       |
| 优化润色（6-8h）   | 摘要独立可读、图表清晰、查重  | 最终版论文      |
| 提交前检查        | 格式/命名/提交渠道      | 确认提交       |


> 系统自动调用 `prompts/04-分路线执行/数学建模全流程.md` + 论文写作辅助

### 知识竞技变体（ACM/蓝桥杯/英语竞赛等）

> 去除建模和论文模块，只走学习+刷题+模考

```
备赛规划 → 知识学习 → 刷题训练 → 模拟实战 → 赛后沉淀
```

#### 备赛规划

**信息收集：** 比赛名称 + 考纲/考点 + 剩余时间 + 当前水平

**输出示例：**

```
蓝桥杯备赛规划（剩余 45 天）
├── 第 1-7 天：基础算法回顾（排序、搜索、贪心）
├── 第 8-14 天：数据结构强化（栈、队列、树、图）
├── 第 15-21 天：常考题型专项（DP、回溯、二分）
├── 第 22-28 天：历年真题刷题（分类刷）
├── 第 29-35 天：模拟考试（计时、查漏补缺）
├── 第 36-42 天：错题重刷 + 薄弱点攻坚
└── 第 43-45 天：全真模拟 + 心态调整
```

#### 知识学习

**模式：** 不限对话数，随时提问

**A/B 路径：**

```
路径 A（零基础）：系统讲解 → 例题 → 练习 → 掌握
路径 B（有基础）：问题定位 → 针对性补充 → 进阶
```

#### 刷题训练

**核心原则：AI 是陪练不是代练**

```
用户拿题 → AI 引导说思路 → 用户说 → AI 指问题 → 用户修正 → AI 确认 → 给最优解做参考
```

**产出：**

- 题目分类清单（按知识模块 + 难度）
- 错题本（错因分类 → 同类题推荐）
- 代码解答归档（自动存入 `06-代码/`）

#### 模拟实战

1. AI 推荐模拟题/用户自选
2. 计时完成，记录每道题用时和得分
3. 输出诊断报告：

```
第三次模考总结
├── 总分：85/150（比上次 +12 分 ↑）
├── 模块正确率：
│   ├── 基础算法 92% ✅
│   ├── 数据结构 78% ⚠️
│   └── 动态规划 40% ❌
├── 时间分析：填空题平均 12min/题，建议控制在 8min 内
└── 下次模考重点：DP 专项训练
```

## 七、通用层 — 赛后沉淀

> 对应 `prompts/06-赛后沉淀/赛后复盘.md`，三个方向共用

```
信息收集 → 过程复盘 → 经验提取 → 简历素材化
```

**信息收集：** 什么比赛 + 结果 + 你的角色 + 自我评价 +（可选）项目文件

**三件事：**


| 步骤    | 产出                           | 用途      |
| ----- | ---------------------------- | ------- |
| 过程复盘  | 时间线 + 3 好 3 改进 + 失败原因分析      | 下次比赛改进  |
| 经验提取  | 3-5 条可迁移通用经验                 | 个人经验库积累 |
| 简历素材化 | 3-5 条 bullet point（不同求职方向版本） | 直接用于求职  |


**约束：**

- 拿了奖也要找问题，没拿奖也要找亮点
- 经验必须可迁移（不是「怎么写申报书」而是「怎么拆复杂任务」）
- bullet point 必须量化

## 九、与旧版（V1、V2）的主要差异


| 维度    | 旧版（V1、V2）       | 优化版 v3                       |
| ----- | --------------- | ---------------------------- |
| 架构    | 方案赛道 / 竞技赛道 双轨  | 🏗️💼📊 三方向 + 通用层            |
| 数模归属  | 横跨两边，定位模糊       | 📊 独立方向，含建模路线 + 知识竞技变体       |
| 入口    | 未明确             | A/B 双路径（知道比赛 / 不知道比赛）        |
| 工具分工  | 列出豆包/秘塔/GPT 等工具 | 系统内部自动调度，用户无感                |
| 流程结构  | 模块平铺，风格不一       | 统一五段式：收集 → 参考 → 执行 → 示例 → 约束 |
| 兜底设计  | 提及但未细化          | 每阶段明确深度/兜底两条路径               |
| 输出标准  | 只写「输出什么」        | 每个模块有具体输出示例                  |
| 提示词映射 | 列在待细化清单         | 直接映射到已入库的 16 个提示词文件          |
| 用户视角  | 需要理解「赛道」概念      | 只需回答「知道比赛吗？」，后续全自动           |
