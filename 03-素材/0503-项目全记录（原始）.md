## 一、项目起源

### 核心痛点（小雷自述）

作为大一新生（湖北师范大学文理学院，网络工程），小雷经历了典型的「信息差」困境：

1. **不知道有什么比赛** — 班级群只发志愿者通知，高价值竞赛信息靠人脉
2. **分不清水赛和硬核赛** — 不知道什么比赛对保研/考研408有用
3. **不知道自己适合什么** — 专业课没学完，怕乱报浪费时间金钱
4. **找队友难** — 想参加比赛但找不到志同道合的人

### 项目定位演变

| 阶段 | 想法 | 判断 |
|------|------|------|
| 初期 | 纯信息聚合网站 | ❌ 任何Agent都能做 |
| 中期 | 双非大学生竞赛/企业机会信息平台 | ✅ 差异化：连接黄石企业 |
| 当前 | 竞赛社交网络+项目孵化器 | ✅ 全链路：选赛→组队→企业对接 |

### 拒绝过的方向（有据可查）

- ❌ 时间管理App（ReFlow）— 市场饱和
- ❌ 工业AI/质检 — 离生活太远，无工厂数据
- ❌ 知识库/Q&A — 其他学校已做
- ❌ 智慧旅游 — 竞赛认可度低
- ❌ 纯概念项目 — 花架子
- ❌ 硬件产品 — 做不了

## 二、技术架构

### V1 CLI 版架构（已完成）

```
用户输入（学校/专业/年级/兴趣）
       ↓
[Python 脚本: match_competitions.py]
       ↓
Tavily 搜索 API ← 生成 6 条搜索关键词
       ↓
去重 → 20条网页结果
       ↓
LLM（Claude Opus 4.7，中转站）→ 两次调用
   ├─ 第1次：匹配「报名中」竞赛 Top5
   └─ 第2次：匹配「已截止」竞赛 Top5
       ↓
格式化输出（终端）
```

### API 依赖

| API | 用途 | 费用 |
|-----|------|------|
| Tavily Search API | 搜索竞赛信息 | 免费额度1000次/月 |
| 中转站（api-siyu.top） | LLM匹配+提取 | 按量计费 |
| Claude Opus 4.7 | 模型 | — |

### 输出字段（V1 最终版）

| 字段 | 说明 | 示例 |
|------|------|------|
| name | 竞赛全称 | 第三届中国高校计算机大赛-AIGC创新赛 |
| match_score | 匹配度(0-100) | 90 |
| match_reason | 匹配理由 | AI+编程高度匹配... |
| participation_type | 参赛形式 | 仅个人 / 仅团队(3-5人) / 个人或团队均可 |
| registration_form | 报名方式 | 个人/团队/学校推荐 |
| registration_url | 报名链接 | https://... |
| registration_deadline | 截止日期 | 2026-10-31 |
| suitable_majors | 适合专业 | 计算机类、网络工程... |
| cross_school_allowed | 是否可跨校 | true/false |
| is_free | 是否免费 | true/false |
| fee_amount | 费用金额 | 免费 / 300元/人 |
| notes | 备注（含金量/难度） | A类竞赛，含金量高... |
| source_url | 来源URL | https://... |
| source_type | 来源类型 | 官方 / 非官方聚合站 |

### 分组显示

- 🟢 **报名中** — 截止日期在当前日期之后，或尚未公布
- 🔴 **已截止** — 已过期但值得关注，供明年规划参考

## 六、设计原则（小雷定下的）

1. **不做好看没用的** — 解决真实痛点，不搞花架子
2. **做自己会用的** — 产品经理必须是用户
3. **先跑通再优化** — CLI 能跑就别急着做网页
4. **用户说需要才算** — 不做自嗨型产品
5. **差异化竞争** — 不和纯信息聚合网站比谁搜得多，而是比谁能「连接人」

| 日期 | 决策 | 理由 |

## 九、开发日志（2026-05-03 完整踩坑记录）

> 记录这一天从零到 V1 完成的所有问题和解决方案，供后续 AI 避免重复踩坑。

### 坑1：Windows CMD GBK 编码报错
- **现象**：`UnicodeEncodeError: 'gbk' codec can't encode character '❌'`
- **原因**：Windows CMD 默认 GBK 编码，不认 emoji（❌、✅、⭐ 等）
- **解决**：在文件头部加 `sys.stdout.reconfigure(encoding="utf-8", errors="replace")`
- **教训**：写 CLI 工具给 Windows 用户，要么不用 emoji，要么开头设 UTF-8

### 坑2：文件路径被转义
- **现象**：`D:\.claude\match_competitions.py` 变成 `D:\.claude\.claudematch_competitions.py`
- **原因**：反斜杠 `\m` 被 Bash 解释为转义字符
- **解决**：用 `D:/.claude/match_competitions.py` 正斜杠 + 正确路径
- **教训**：跨平台跑脚本，路径用正斜杠 `/`

### 坑3：环境变量跨命令不传递
- **现象**：`set KEY=value && python script.py` 中 Python 读不到环境变量
- **原因**：Bash 工具不是 cmd.exe，`set` 命令语义不同
- **最终方案**：Bash 语法 `TAVILY_API_KEY="xxx" RELAY_API_KEY="xxx" python script.py`
- **教训**：Bash 工具运行在 bash/zsh 环境，不能用 cmd 语法

### 坑4：API SDK 反复切换
- **初始**：OpenAI SDK（`from openai import OpenAI`）
- **第1次切换**：用户说用 Claude，改为 Anthropic SDK
- **第2次切换**：用户说 Key 是「中转站」不是直连，只有 OpenAI 兼容格式
- **最终方案**：用 OpenAI SDK + 自定义 `base_url="https://api-siyu.top/v1"`，model 填 `claude-opus-4-7`
- **教训**：先确认 API 类型（直连/中转站/原生格式）再选 SDK，避免来回改

### 坑5：LLM 输出被截断（最折腾的坑）
- **现象**：`json.loads()` 报错，JSON 在 `"source_url": "https://` 处截断
- **原因**：中转站对单次请求有隐藏的输出长度限制
- **最终方案**：拆成两次 LLM 调用（open + closed），数据量减半
- **附加防护**：`result_text.rfind("]")` 找到最后一个右括号，截掉后面不完整的数据

### 坑6：同一个比赛在两个分组里重复出现
- **当前状态**：未修复，视为可接受的小瑕疵
- **建议后续修复**：在 `print_results` 中对两个列表做 URL 去重

### 坑7：sohu.com 被误标为「官方」
- **当前状态**：已修 prompt，部分改善但非 100%
- **建议后续修复**：在代码层做域名白名单，不在 prompt 里猜

### 坑8：用户反复改需求（正常，但需要记录）
- 功能迭代是正常的，**V2 做网页版时要设计可扩展的数据模型**，不要每次加字段就改 prompt。

### 当前已知问题（未修复）

| 问题 | 严重度 | 备注 |
|------|--------|------|
| 同一比赛出现在两个分组 | 低 | URL 去重可修 |
| 来源类型偶尔误判 | 低 | 代码层域名白名单可修 |
| 费用信息大多数显示「免费」 | 中 | 搜索结果缺少付费信息，数据源问题 |
| 截止日期大量显示「未知」 | 高 | 竞赛网站很少写明确截止日期，数据源根本性问题 |
| 搜索 API 只能搜公开网页 | 高 | 班级群/辅导员朋友圈的竞赛通知搜不到 |

### 如果你换一个 AI 继续开发，告诉它的三句话

1. **不要改架构**：搜索API + LLM匹配 这条路验证过了，能用。别换成爬虫。
2. **先修那个「截止日期未知」的问题**：这是最大短板，可以考虑加一个「预计报名时间」字段，用往年规律推测。
3. **网页版从 Flask 起步**：后端一个路由 `/api/match`，前端一个表单。别上 React/Vue，太早了。
